# 🛟 Бот поддержки (FindUP)

Телеграм-бот для поддержки: пользователь пишет боту — обращение попадает к админу
в виде **тикета**. Управление полностью **внутри Telegram**: фильтры по статусам,
список тикетов, история переписки, ответы в один клик.

## Возможности

- 📥 Любое сообщение пользователя (текст, фото, файл, голосовое, видео) создаёт
  или дополняет тикет.
- 🔔 Админ мгновенно получает уведомление с карточкой обращения.
- 🗂 **Тикет-система** с тремя статусами:
  - 🆕 Ожидает ответа — пользователь написал, ответа ещё нет;
  - ✉️ Отвечено — оператор ответил;
  - 🔒 Закрыто — обращение завершено.
- 🔎 Поиск по ID, @username и тексту сообщений.
- ✍️ Ответ двумя способами: кнопкой «Ответить» или обычным **reply** на уведомление.
- 📜 Полная история переписки по каждому тикету.
- 📊 Статистика.
- 👥 Поддержка нескольких админов (`ADMIN_IDS` через запятую).

## Быстрый старт

```bash
cd "ТП FindUP"
./run.sh
```

Скрипт сам создаст виртуальное окружение, установит зависимости и запустит бота.

### Вручную

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
python bot.py
```

## Настройка

Всё в файле `.env` (см. `.env.example`):

| Переменная | Значение |
|------------|----------|
| `BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather) |
| `ADMIN_IDS` | ID админов через запятую. Свой ID — у [@userinfobot](https://t.me/userinfobot) |
| `DB_PATH`   | Путь к базе SQLite (по умолчанию `support.db`) |
| `PAGE_SIZE` | Сколько тикетов на странице списка (по умолчанию 6) |

Токен и ID администратора уже прописаны в `.env`.

## Как пользоваться

**Пользователь:** открывает бота, пишет вопрос. Получает подтверждение и ответы
оператора прямо в чате.

**Админ:** команда `/panel` (или `/start`) открывает панель. Дальше — кнопками:
выбираете статус → тикет → «Ответить». Либо просто отвечаете (reply) на
пришедшее уведомление.

Команды админа: `/panel`, `/menu`, `/cancel`.

## Структура

| Файл | Назначение |
|------|-----------|
| `bot.py` | запуск, сборка диспетчера |
| `config.py` | чтение `.env` |
| `db.py` | работа с SQLite (тикеты, сообщения) |
| `texts.py` | тексты и форматирование, разбор сообщений |
| `keyboards.py` | inline-клавиатуры |
| `user_handlers.py` | сообщения от пользователей |
| `admin_handlers.py` | админ-панель |

## Заметки

- Хранилище — SQLite (`support.db`), создаётся автоматически.
- Транспорт — long polling, хостинг/вебхук не нужны. Бот работает, пока запущен
  процесс. Для постоянной работы держите его на сервере (например, через
  `systemd`, `screen`/`tmux` или `pm2`).
- `.env` и `support.db` добавлены в `.gitignore` — токен и переписка не попадут в git.
